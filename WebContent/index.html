Project1

지오짐(GeoGym)

가상의 중/대형 규모의 헬스장을 상정하고
홍보와 운영을 겸한 헬스장 홈페이지 및 솔루션을 구축한다.

개발환경

운영체제 : window10, Debian
서버 : Apache Tomcat 9.0
DBMS : SQLite
프레임워크 : Spring MVC framework 5.2.2, Spring Tool Suite 3, Bootstrap
라이브러리 : jQuery, AJAX
개발언어 : JAVA(JDK1.8), HTML5/CSS, JavaScript, JSP/Servlet, SQL
개발도구 : Eclipse, eXERD, draw.io, Git, DBbrowser


구현기능

1. 로그인(구글로 로그인) / 회원가입(정규식, 비밀번호 암호화, 이메일 인증)
2. QnA 게시판(공개여부설정, 댓글, 답변 조회수 노출, 검색 가능)
3. 마이페이지(개인프로필과 신체정보 노출 및 구글차트API를 활용한 신체 변화량 노출)
4. 트레이너 전용 담당회원 관리 페이지
5. 결제기능(카카오페이 API)
6. 달력을 이용한 스케쥴 관리
7. 알림 기능
8. 관리자 페이지(소속 트레이너 스케쥴 관리, 회원 정보 관리 등)
9. 구글 지도 API
10. 반응형웹 구현(부트스트랩 이용)


담당역할

1. 마이페이지
2. 트레이너전용 담당회원 관리 페이지
3. 반응형웹(담당페이지만)
4. QnA 게시판
5. 파일 업로드 / 다운로드


참여소감

JSP/servlet기반 구현 작업에 적응하기가 무섭게 새로운 개발 환경인 스프링 프레임워크에 또 다시 적응하느라 구현 초반에 애를 먹기도 했지만 점차 스프링 사용에 익숙해 지게 되었고 처음 접해보는 SQLite나 LocalDate타입 등 훈련과정동안 배웠던 기술 외에도 여러 가지 기술을 접해 볼수 있어서 좋았다.

Project1_Storyboards

마이페이지_신체정보관리
화면 설명

첫번째 : 회원이면 누구나 사용 가능한 마이페이지
두번째 : 트레이너 전용 담당회원 관리 페이지
세번째 : 첨부한 inbody 사진 팝업
네번째 : 신체정보입력 폼 및 파일첨부 팝업
구현기능설명

1. 업로드한 프로필사진 노출( commons_fileupload 사용 )
2. 회원 개인 정보 노출
3. 신체 변화량 7일단위 / 30일단위로 노출. LocalDate타입을 사용해 선택 단위마다 한건씩 조회해 차트에 반영( 구글차트API사용 ) 해당기능 주요코드
4. 조회된 결과가 차트로 노출됨. 키 정보는 slide토글을 사용해 가리기 가능
5. 트레이너가 담당중인 회원의 프로필 사진 노출( commons_fileupload 사용 )
6. 담담중인 회원의 신체정보외에도 해당 회원의 특이사항 노출( 트레이너만 확인 가능 )
7. 트레이너가 첨부한 담당회원의 inbody 사진 팝업으로 노출
8. 신체정보 입력 폼( 하루에 한번만 입력 가능 )
9. 회원의 특이사항 입력 폼( 트레이너만 입력 및 조회가 가능 / 트레이너일 경우에만 노출 )
10. 해당 회원 inbody 사진 업로드 가능( 하루에 한번만 입력 가능 )

게시판
화면 설명

상단 : QnA 게시판 게시글 목록
중단 : 게시글 상세보기
하단 : 게시글 작성
구현기능설명

1. 한 페이지당 10개씩 게시글 노출

3. 질문글 작성시 공개글로 설정했을 경우
4. 10페이지씩 이동 가능하도록 페이지네이션 구현
5. 제목, 작성자, 내용으로 검색 가능
6. 글 작성자일 경우에만 수정 / 삭제 가능
7. 사이트 관리자 or 트레이너일 경우 댓글 형식으로 답글 작성 가능
8. 답변 작성시 사진 첨부 가능
9. CKEditor로 게시글 작성 구현
10. 질문글 작성시에도 사진 첨부 가능
11. 글 작성시 공개여부 설정 가능

Project2

방캅스(BangCops)

고객이 참여하고, 신고할 수 있는 허위 매물 근절 웹사이트를 구축한다.

개발환경

운영체제 : window10
서버 : Apache Tomcat 9.0
DBMS : oracle 11g
프레임워크 : Bootstrap
라이브러리 : jQuery, AJAX
개발언어 : JAVA(JDK1.8), HTML5/CSS, JavaScript, JSP/Servlet, SQL
개발도구 : Eclipse, eXERD, draw.io, Git, sqldeveloper


구현기능

1. 구글 지도 API
2. 구글 차트 API
3. 키워드를 포함한 뉴스기사 크롤링
4. 회원가입( 정규식, 이메일 인증 ) / 로그인
5. 게시판( 목록, 작성, 검색, 수정, 삭제, 조회수, 추천, 댓글 )
6. 관리자 페이지( 회원 목록, 게시판 목록, 다중삭제 )
7. 마이페이지( 모아보기, 정보수정 )
8. 고객상담 1:1채팅 API (채널톡)


담당역할

1. 게시판( 자유,후기,공지사항,FAQ )
2. 메인페이지
3. 파일 업로드 / 다운로드


참여소감

첫 프로젝트였는데 구현을 막 시작했을때는 어려웠지만 하다보니 적응이 되었다. 기획했던 만큼 구현을 다 할 수 있을지 걱정했었는데 빠지는 부분 없이 구현되어서 다행이였고, MVC 패턴을 사용해 구현시 데이터가 어떤 흐름으로 전달되는지 알 수 있어서 좋았다. 어려워했던 AJAX 사용도 잘 할 수 있게돼 자신감이 생겼다.

메인페이지
화면 설명

구현기능설명

커뮤니티
화면 설명

상단 : 사이트 전체의 커뮤니티 현황을 볼 수 있는 커뮤니티 메인페이지
하단 : 게시글 목록 페이지와 상세보기 페이지
구현기능설명

1. 각 게시판마다 key값을 부여해 여러 게시판을 쉽게 구현 할 수 있도록 함
2. 공지사항은 최신순으로 상위 다섯개를 노출하고 자유게시판, 이용후기게시판은 추천순으로 다섯개 노출
3. 게시판목록은 20개씩 나열하도록 구현
4. 추천은 추천한 회원의 ID로 COUNT하여 중복 추천이 불가능 하도록 구현
5. 제목, 작성자, 내용으로 검색 가능
6. 현재 페이지 기준 최대 10개의 페이지로 이동할 수 있게 페이지네이션 구현
7. 추천은 AJAX를 사용하여 추천시 추천 해제가 가능하도록 구현
8. 글 작성시 파일첨부 가능 및 파일 다운로드도 가능(commons-fileupload 사용)
9. 게시글 작성자일 경우에만 수정 / 삭제 접근 가능
