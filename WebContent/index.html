My Character

단점은 일상적인 루틴에서 벗어난 예상치 못한 일이 발생하거나 한번도 경험하지 못한 일이 발생하는 경우에는 당황하는 경황이 있습니다. 물론 경력이 쌓이고 여러 경우를 겪다 보면 자연스럽게 해결될 문제지만 어떤 일을 하더라도 100% 예방하기는 힘들다는 것을 잘 알고있기 때문에 이러한 일이 생길 경우 당황하지 않으려 가능한 최대한의 경우의 수를 미리 파악해 저만의 Check List를 작성하는 습관을 들여 문제 해결에 노력하고 있습니다.

Project1

지오짐(GeoGym)

가상의 중/대형 규모의 헬스장을 상정하고
홍보와 운영을 겸한 헬스장 홈페이지 및 솔루션을 구축한다.
개발환경

운영체제 : window10, Debian
서버 : Apache Tomcat 9.0
DBMS : SQLite
프레임워크 : Spring MVC framework 5.2.2, Spring Tool Suite 3, Bootstrap
라이브러리 : jQuery, AJAX
개발언어 : JAVA(JDK1.8), HTML5/CSS, JavaScript, JSP/Servlet, SQL
개발도구 : Eclipse, eXERD, draw.io, Git

구현기능

1. 로그인(구글로 로그인) / 회원가입(정규식, 비밀번호 암호화, 이메일 인증)
2. QnA 게시판(공개여부설정, 댓글, 답변 조회수 노출, 검색 가능)
3. 마이페이지(개인프로필과 신체정보 노출 및 구글차트API를 활용한 신체 변화량 노출)
4. 트레이너 전용 담당회원 관리 페이지
5. 결제기능(카카오페이 API)
6. 달력을 이용한 스케쥴 관리
7. 알림 기능
8. 관리자 페이지(소속 트레이너 스케쥴 관리, 회원 정보 관리 등)
9. 구글 지도 API
10. 반응형웹 구현(부트스트랩 이용)

DB설계

담당역할

1. 마이페이지
2. 트레이너전용 담당회원 관리 페이지
3. 반응형웹(담당페이지만)
4. QnA 게시판
5. 파일 업로드 / 다운로드

화면 설명

회원이면 누구나 사용 가능한 마이페이지(좌측)와 트레이너 전용 담당회원 관리 페이지(우측)
구현기능설명

1. 클라이언트나 트레이너는 자신의 신체정보 입력 가능( 하루에 한번만 제한 )

구현기능설명

1. 현재 사용중인 디스플레이 해상도에 맞춰 반응하는 반응형 웹 구현

화면 설명

사이트 회원들의 질문을 묻고 답할 수 있는 QnA 게시판
구현기능설명

1. 한 페이지당 10개씩 게시글이 노출 되도록 페이징 구현
2. 글 작성시 공개여부 설정 가능

Project2

방캅스(BangCops)

고객이 참여하고, 신고할 수 있는 허위 매물 근절 웹사이트를 구축한다.
개발환경

운영체제 : window10
서버 : Apache Tomcat 9.0
DBMS : oracle 11g
프레임워크 : Spring MVC framework 5.2.2, Spring Tool Suite 3, Bootstrap
라이브러리 : jQuery, AJAX
개발언어 : JAVA(JDK1.8), HTML5/CSS, JavaScript, JSP/Servlet, SQL
개발도구 : Eclipse, eXERD, draw.io, Git, sqldeveloper

구현기능

1. 구글 지도 API
2. 구글 차트 API
3. 키워드를 포함한 뉴스기사 크롤링
4. 회원가입( 정규식, 이메일 인증 ) / 로그인
5. 게시판( 목록, 작성, 검색, 수정, 삭제, 조회수, 추천, 댓글 )
6. 관리자 페이지( 회원 목록, 게시판 목록, 다중삭제 )
7. 마이페이지( 모아보기, 정보수정 )
8. 고객상담 1:1채팅 API (채널톡)

DB설계

담당역할

1. 게시판( 자유,후기,공지사항,FAQ )
2. 메인페이지
3. 파일 업로드 / 다운로드

담당 구현 페이지
화면 설명

사이트 메인페이지 및 하단 통계
구현기능설명

1. 시/구/상세주소로 검색 가능하게 구현
2. 검색창 하단 통계(오늘의 신고건수, 전체 신고건수) 노출
3. 검색시 신고 게시판의 검색 List로 자동으로 이동 후 결과 노출
4. 구 별 신고건수 현황은 구글 차트 API를 활용하여 구현함. 신고테이블을 구 별로 GROUP BY 하여 COUNT 한 결과가 차트에 반영됨

화면 설명

각 게시판으로 이동 하기 전에 사이트 전체의 커뮤니티 현황을 볼 수 있는 커뮤니티 메인페이지
구현기능설명

1. 각 게시판의 게시글 목록을 다섯 개씩 노출하도록 구현
2. 공지사항 - 최신순,
자유게시판, 이용후기게시판 - 추천순으로 노출

화면 설명

게시판 게시글 목록 및 게시글 상세보기
구현기능설명

1. 게시글 목록에 번호, 제목, 작성자, 작성일자, 조회수, 추천수 노출
2. 목록을 20개씩 나열하도록 페이징 구현, 제목 옆에 댓글수 표시
3. 게시글 추천시 페이지가 새로고침되지 않도록 AJAX를 사용해 구현
4. 게시글 작성시 파일 업로드 가능 (제한 용량 10MB로 설정해 구현)
5. 게시글 작성자일 경우 게시글 수정, 삭제 가능
6. 댓글기능 구현, 댓글이 나타나는 부분은 슬라이드토글을 사용해 가리기 가능
7. 검색은 제목, 작성자, 내용으로 검색 가능

참여소감

첫 프로젝트였는데 구현을 막 시작했을때는 어려웠지만 하다보니 적응이 되었다. 기획했던 만큼 구현을 다 할 수 있을지 걱정했었는데 빠지는 부분 없이 구현되어서 다행이였고, MVC 패턴을 사용해 구현시 데이터가 어떤 흐름으로 전달되는지 알 수 있어서 좋았다. 어려워했던 AJAX 사용도 잘 할 수 있게돼 자신감이 생겼다.
